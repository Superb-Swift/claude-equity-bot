## How new chats work (why this manifest exists)

Claude's workspace **resets between conversations**. Memory carries the project's *story* (phases, verdicts, constraints, conventions) but **not file contents** — so every new chat needs its working files uploaded fresh (or pulled from the repo, Option B below). Within a single conversation, files persist across messages; you only re-upload when data changes (e.g., a new day's tracker).

## Option B — repo pull instead of code uploads

The repo is public (`Superb-Swift/claude-equity-bot`) and Claude's sandbox can reach GitHub, so for **code files** you can say "pull from the repo" instead of uploading — *if you've pushed first*. Caveats: uploaded files always win as source-of-truth (the repo lags your working tree between pushes); data files (tracker, day logs) should still be uploaded directly since they change daily and may not belong in the repo.

## Repo housekeeping (recommended, from the 7/6 file-state screenshots)

1. **Duplicate hypothesis MDs, two generations apart** — `docs/` holds 6/13–6/14 copies of `h1_lag_trace_WMT`, `h2_…baseline`, `h3_…portfolio`, `h4_…baseline`; root holds the fresh 7/6 copies (weekly-review outputs). Pick one home: **root = live outputs** (regenerated by `run_weekly_review.bat`), **docs/Phase 3-A docs/ = frozen closeout snapshots**. Move the current root MDs into the archive folder once, then let root regenerate freely.
2. **Archive the 3-A era into `docs/Phase 3-A docs/`** (mirroring your `Phase 2 docs` pattern): Worksheets **v1 + v2 (superseded)**, **v3 (SIGNED — the instrument of record; consider exporting a PDF copy or marking read-only)**, `Phase3A_Closeout_Memo.md`, `Phase3A_Kickoff_Prompt.md`, both `Daily_Workflow_v3_x_Phase3A` docs, final `regime_stratification.md`, and the closeout-day `near_miss_registry.md` snapshot.
3. **`Phase3B_Kickoff_Prompt.md` lives only in root** — copy it into `docs/` so the docs folder stays the complete document index.
4. **Retired per the closeout**: `run_ab_test.bat` (A/B terminated) — move to an archive folder or delete; same for `logs/TEST signals_2026-06-15.txt` and `claude_equity_bot_tracker_pre_cohort_tabs.xlsx` (pre-cohort-era tracker).
5. **Then push** — the closeout bundle (memo, kickoff, signed v3, notes, stratification, reorganized folders) makes a clean "Phase 3-A closed" commit.
